Problems Encountered in the Map OSM Data¶

Main problems encountered are:

  • Street names are over-abbreviated and not consistent.
  • Postcode are incorrect or contain information that is not a postcode.

The sections below describe what these problems are in detail.

Auditing street names show the following common issues in the original osm file:

  1. Many street types are abbreviated, stored as all upper case or lower case. For example, "Road" is shown as "Rd.", "Rd" or "road".
  2. Directions are shown in various format: "N", "N.", "North", etc.
  3. Street names with numbers in it are in various format. For example, interstate highway names are displayed as "I 30", "Interstate Highway 30", etc.
  4. "Suite" in the street name is displayed inconsistently as "Suite", "suite", "#", etc. It should be converted to "Suite".

There are are multiple "special" issues with the street names:

  1. Street type is missing from the name. For example: "Arapaho" should be "Arapaho Road"
  2. "v" value is not a street name, but rather a house number or phone number
  3. "v" value contains not only street name but also other information

These special cases are first checked using the "id" attribute on openstreet map website. For example, if the node tag id is 1234567, its data can be viewed by typing "https://www.openstreetmap.org/node/1234567" in web browser.

These issues are fixed by running "clean_osm.py" to generate a new osm file and this file is used to convert to json foramt.

Data Cleaning and Conversion¶

Data Cleaning -- Generating a cleaned OSM data file¶

The street name and postcode have some special errors that need to be handled mannually. For example, the postcode field may not contain any valid post code. In these cases, the "id" value of the corresponding "way" or "node" tag is found first. Then the data is checked through the openstreet map website by typing "https://www.openstreetmap.org//" in a web browser. The correct information is gathered and then fixed by running "clean_osm.py".
Other issues such as over-abbreviated word in street names (type, direction, etc.) are also fixed through "clean_osm.py".
Changes made are printed to the screen while running the script.

Converting OSM data file to JSON format¶

In [8]: 
# Converting data to json format
JSON_FILE = "../data/dallas_texas.json"
%run osm_to_json.py $NEW_OSM_FILE $JSON_FILE

Note:

The second level tag "tag" also has attribute "type". "type" is reserved for json document types, i.e. node, way. To avoid overwriting the document type, "type" attribute in the second-level tag is renamed to "tag_type".
The created json file is then imported into database "osm" collection "dfw" in MongoDB using:
mongoimport -d osm -c dfw --file ../data/dallas_texas.json
